So, basically this is a really awesome project that recives the data from a pulse sensor to an app (The UI sucks btw)
The sensor uses photoplethysmography to measure blood flow changes.
Then the sensor out puts that into a serial value 😱😱😱😱😱😱
The arduino code converts that value into an estimated (emphasis on the estimated) BMP.
While doing this project I learned about this really awesome thing called WebSockets.
It uses real-time, low-latency, bidirectional data which is extanged between servers. (I told you it was cool 😎)
In short ts justs transfer the sensor data to the app.
There is not much to say about the app other than that it uses Swift (SORRY LOSERS!!! wait... I mean Android users 🤪)
It just shows the user their heart rate :)
